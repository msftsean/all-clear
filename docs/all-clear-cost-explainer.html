The five words everything below uses

If we agree on these, the dollars stop being confusing.

🎫Seat
One person, running the lab one time. A "seat" = a developer (a participant) for a single lab day. 30 developers doing the lab once = 30 seats. The same person doing it twice = 2 seats of usage.
📡Signal
One thing the app triages — a report, a call, an alert ("downed line at 5th & Main"). This is the unit that burns tokens. A seat sends many signals during a lab.
🪙Token
The billing atom for the AI model (~¾ of a word). UBB charges per token. All our runtime numbers roll up from tokens-per-signal.
📅Lab day
One full run of the 6.5-hour build lab by one seat. Most of it runs offline (mock mode = $0 tokens); cost only accrues when a seat fires live signals.
⛽The two meters
Run it (Azure tokens, UBB) and Build it (GitHub Copilot seats). Different bills, different teams, never the same line item. The next slide is entirely about this.
🎟️Credit
A prepaid unit — in two flavors that aren't interchangeable. An Azure credit = $1 of token usage (buys Run it). A GitHub Copilot credit = one premium request — a single AI interaction × the model multiplier (buys Build it).

The #1 thing to get right

There are two separate meters. Don't conflate them.

⛽ Run it

Azure OpenAI · pay-per-token · UBB
  • The tokens the app burns when it triages a signal.
  • Metered per token on Azure — this is the UBB line item.
  • Tiny: ~2,500 tokens per signal (~$0.004).
  • Who owns it: your Azure subscription.
Scales with usage (how many signals get triaged).

🛠️ Build it

GitHub Copilot · per-seat license
  • The tokens the coding agent burned to build & evolve the app.
  • Billed as Copilot seats + premium requests — not Azure UBB.
  • Big number (~108M tokens) but it's a seat, not a token bill.
  • Who owns it: your dev team's Copilot license.
Scales with developers (how many people build/extend).

Lily's question · what do the credits mean?

Two kinds of credit. Only one buys tokens.

"Credit" is overloaded. Which meter it pays comes down to what it's denominated in — dollars or requests.

💵 Azure credit

Sponsorship · MACC · free trial — denominated in $
  • It's dollars, and Azure OpenAI bills dollars per token.
  • So it maps 1 : 1 onto the "Run it" token bill.
  • $1 ≈ 250 signals ≈ 0.25M tokens.
  • The only credit that pays the app's runtime.
Pays for: running the app (Azure tokens).

🎟️ GitHub Copilot credit

Premium request — denominated in interactions
  • It's a request, not a token. 1 credit = 1 AI interaction.
  • Each interaction × a model multiplier (0.33× → 57×).
  • Included per seat: 300 Business / 1,000 Enterprise; overage $0.04/req.
  • Pays to build the app — and to run Copilot inside the lab.
Pays for: building & in-lab Copilot — never Azure runtime.
✕

There is no clean "1 credit = N tokens." A premium request can be any size. A GitHub credit buys an interaction; an Azure credit buys tokens. Don't pay runtime out of Copilot, or building out of Azure.

Honest fine print

Four things to say out loud

🎙️
Voice is optional and OFF by default.

Realtime audio (gpt-realtime) is pricier than text. If you turn it on, budget ~$2–5 per spoken demo — a separate line, not part of the per-seat number.

🏢
Fixed infrastructure ≠ tokens.

The app's standby footprint (11 Azure services) is the app's own estimate of ~$1,555 / month, independent of how many signals run. That's hosting, not UBB usage.

💵
Pricing basis.

gpt-5.1 at $1.25/$10 per 1M in/out; embeddings $0.02/1M. Public PAYG rates; your EA/MACC discount only makes it cheaper.

📏
The build number is a floor.

~108M tokens = measured Copilot CLI work over 45 days. It undercounts (excludes other tools, other devs, older history) — but it's billed by seat regardless.

⚖️
Credits are mid-migration.

GitHub is moving from request-based "credits" toward usage-based (token) billing; Azure is already per-token. Multipliers shown are the request-based model. Either way the rule holds: GitHub pays to build, Azure pays to run.
