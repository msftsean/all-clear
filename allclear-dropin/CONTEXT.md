# CONTEXT.md: All Clear Domain Language

Ubiquitous language for All Clear. One canonical term per concept. Coding agents and humans use these words exactly as defined here. If code and this document disagree, this document wins and the code gets fixed.

## Core Domain Objects

**Signal.** A single inbound communication on any channel: one chat message thread, one voice call, one phone call, one submitted report. Signals are never deduplicated away; every signal is preserved and attributable. A signal is what arrives.

**Incident.** The underlying real-world event that signals describe. One downed power line is one incident, even if 40 people call about it. Incidents have an id (AC-####), a severity, a queue, an SLA clock, a magnitude, and a status. An incident is what is happening.

**Report.** The association of a signal to an incident. When dedup attaches a signal to an existing incident, that attachment is a report and the incident's magnitude increments.

**Magnitude.** The count of reports attached to an incident. A proxy for scale and public impact. Drives ClearBoard visual weight.

**Queue.** The destination work stream for an incident (field-operations, customer-comms, compliance-desk, engineering, escalations). Replaces 47 Doors' "department."

**Sitrep.** Situation report. A citation-grounded summary of an incident generated by ActionAgent's generate_sitrep tool. Every factual claim cites a source record.

**All clear.** The terminal state: every open incident resolved, every SLA clock satisfied. The ClearBoard flips to its green all-clear state. Also the product name.

## Pipeline Vocabulary

**QueryAgent.** MAF agent. Classifies one signal into a SignalClassification (intent, entities, severity indicators, PII flags). Authority: classify only. It cannot route, create, or search.

**RouterExecutor.** Deterministic MAF workflow executor. Zero LLM calls, by design and by test. Performs dedup, then severity/SLA mapping, then escalation rules. Produces a RoutingDecision.

**ActionAgent.** MAF agent with exactly three tools: create_incident, search_knowledge, generate_sitrep. Authority: only what its tools permit. It cannot approve waivers, modify records outside its tools, or bypass escalation.

**Dedup.** Embedding-similarity comparison of an inbound signal against open incidents in the same intent_category. At or above DEDUP_THRESHOLD (config, default 0.83 cosine): ATTACH_TO_INCIDENT. Below: OPEN_INCIDENT.

**ATTACH_TO_INCIDENT.** RoutingDecision outcome: signal becomes a report on an existing incident, magnitude increments, reporter receives a short acknowledgment. No knowledge search on this path (keeps surge latency flat).

**OPEN_INCIDENT.** RoutingDecision outcome: a new incident is created and the full ActionAgent path runs (create, search knowledge, respond with citations).

**Escalation.** Mandatory handoff to a human queue. Triggered by the escalation intent set, PII exposure rules, or sentiment/safety rules. Escalation logic is a safety control: code that weakens it is a security blocker, not a refactor.

**Surge.** An inbound volume spike (storm, outage, recall) where most signals are duplicates of few incidents. The hero scenario. Exercised by the surge replay fixtures.

**ClearBoard.** The live frontend view: react-leaflet map with incident pins that visibly merge as dedup attaches reports, magnitude counters, SLA breach highlighting, and the all-clear green state.

## Severity and SLA

**Severity.** SEV1 through SEV4, SEV1 most severe. Mapped from classification severity indicators by RouterExecutor rules, never by model vibes. Customer-support ISVs read these as P1-P4; same scale.

- **SEV1:** life safety, total outage, statutory breach clock running. Immediate escalation, 15-minute response SLA.
- **SEV2:** major impairment, public-facing impact, spreading. 1-hour response SLA.
- **SEV3:** contained, single-party impact. 4-hour response SLA.
- **SEV4:** informational, routine request. Next-business-day SLA.

**SLA clock.** Started at incident creation, per severity. A breached clock highlights on the ClearBoard and is recorded in the audit log.

**Statutory clock.** A legally mandated deadline (breach notification windows, recall reporting). Treated as SEV1-forcing regardless of other indicators.

## Industry Terms (the three ISV verticals)

**CAD.** Computer-Aided Dispatch. The system of record in public safety dispatch centers. All Clear is CAD-adjacent triage, not a CAD replacement.

**PSAP.** Public Safety Answering Point. Where emergency calls land. The non-emergency overflow problem PSAPs face is the public-safety skin of the hero scenario.

**NFIRS / NIBRS.** National Fire Incident Reporting System / National Incident-Based Reporting System. Compliance report formats fire and law enforcement must file. The regulated-industry payoff of generate_sitrep.

**CJIS.** Criminal Justice Information Services security policy. The data-discipline mindset for this codebase: least privilege, full audit, no PII echo. We adopt the mindset everywhere even where CJIS does not legally apply.

**Unit status codes.** Field unit states in dispatch: en route, on scene, 10-8 (available), 10-7 (out of service). Appear in sample data and SOPs.

**Tier-0 deflection.** Customer support term: resolving an inquiry before it consumes human agent time. The customer-support skin of the pipeline.

## Engineering Conventions

**allclear.** The one-word lowercase slug for packages, env vars, and CLI. The repo is EstablishedCorp/all-clear; Python packages cannot contain hyphens, so code uses allclear.

**Mock mode.** USE_MOCK_MODE=true runs the entire pipeline offline against mock twins of every Azure service. Every live service has a mock twin and they stay in lockstep.

**Verifier.** A test, eval set, fixture, or checkpoint script with an executable command and exit code. Built before the implementation it grades (Loop Protocol, specs/001-maf-rehost/plan.md). Owned by Barton; never edited by implementers.

**Bounded authority.** Each agent can do only what its role permits, enforced by code structure (tools, interfaces) rather than prompt hope.
