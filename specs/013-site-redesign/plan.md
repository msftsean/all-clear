# Implementation Plan: Public SWA Site Redesign

**Branch**: `013-site-redesign` | **Date**: 2026-06-01 | **Spec**: ./spec.md

## Summary

Restyle the three live public React/Vite/Tailwind sites (workshop, coach, hackathon) so each fits its audience and passes `npx impeccable detect <site>/src` with **0 anti-patterns**, while keeping all content, tests, accessibility, and deploy pipelines intact. Presentation-only changes (Tailwind theme tokens, component class treatments, CSS, fonts). Delivered incrementally, one site per PR, in priority order P1 workshop → P2 coach → P3 hackathon. Per-site target aesthetics are captured as DESIGN.md briefs in `contracts/design-briefs.md`.

## Technical Context

**Language/Version**: TypeScript 5, React 18, Node 18+
**Primary Dependencies**: Vite 5, Tailwind CSS 3.4, @heroicons/react
**Testing**: Playwright e2e (per site), `npm run build`, `npm run typecheck` (coach), `npx impeccable detect` (design gate)
**Target Platform**: Azure Static Web Apps (static bundle)
**Project Type**: Multiple independent static front-end apps in one repo
**Performance Goals**: No bundle-size regression beyond ~10%; preserve existing build times
**Constraints**: Presentation-only — no rendered-text/content changes; WCAG AA; no cross-site/backend edits; each site's `staticwebapp.config.json` + deploy workflow unchanged
**Scale/Scope**: 3 sites; baseline Impeccable findings 30 / 3 / 1

## Constitution Check

The constitution (v1.1.0) is agent/voice-focused; for these static marketing/reference sites only two principles bind:

- **Principle V (Test-First, light)**: each site already has Playwright e2e + build; the Impeccable detector is added as an objective design gate. No new runtime behavior, so no new unit tests required beyond keeping existing suites green and asserting detect=0.
- **Principle VI (Accessibility / WCAG AA)**: redesign MUST preserve or improve contrast, keyboard nav, focus-visible, single-main landmark, skip link. coach-site's existing a11y e2e tests guard this.

**Result: PASS** — no violations; no complexity deviations.

## Approach

For each site (in priority order):

1. **Brief**: apply the site's DESIGN.md from `contracts/design-briefs.md` (Refero-anchored palette/type/spacing/component treatments).
2. **Tokens**: update `tailwind.config.js` theme (colors, fontFamily) and `src/index.css` (base type, focus, fonts) per the brief.
3. **Components**: replace every `border-l-4` side-tab with the brief's accent treatment (hairline border / soft fill / top rule / heading emphasis); apply spacing + elevation tokens; fix flagged contrast pairs.
4. **Gate**: `npx impeccable detect <site>/src` MUST be 0; iterate until clean.
5. **Verify**: `npm run typecheck` (where present) + `npm run build` + `npx playwright test` all green; confirm no rendered-text diffs (content/`source:` unchanged).
6. **Ship**: PR for that site only → CI green → squash-merge → site deploys via its existing workflow → curl-verify still 200 (or 302 for gated, N/A here).

### The Impeccable detector as the objective gate

`npx impeccable detect` is deterministic (no LLM, no API key) and runs in CI-friendly fashion (exit 2 on findings). It is the SC-001 acceptance gate. The redesign edits are authored directly per the briefs; the detector + existing tests + manual contrast review form the guardrails. (The interactive `impeccable skills install` is intentionally NOT used, to avoid committing third-party skill files into `.claude/.agents/.github`.)

## Risks & Mitigations

- **Risk**: removing `border-l-4` flattens visual hierarchy. **Mitigation**: replace with heading weight, soft-fill callouts, or a 1px top rule that preserves emphasis (validated by eye + e2e that key elements still render).
- **Risk**: new palette regresses contrast. **Mitigation**: choose AA-checked token pairs; coach a11y e2e + manual contrast check.
- **Risk**: accidental content change. **Mitigation**: presentation-only edits; diff review asserts no text nodes change; coach content-drift test must pass.
- **Risk**: detector flags a *new* tell (e.g., gradient text, Inter). **Mitigation**: iterate to 0 before shipping; prefer non-Inter humanist faces.
